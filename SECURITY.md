# Security Policy

## Поддерживаемые версии

| Версия | Поддержка |
|--------|-----------|
| 1.x.x  | ✅ Да     |

## Сообщение о уязвимостях

Мы высоко ценим ваш вклад в безопасность проекта Pingcheck. Если вы обнаружили уязвимость, пожалуйста, следуйте этим инструкциям:

### Как сообщить

- **Email**: nicholas.rogov@ya.ru
- **Язык**: русский/английский

### Что включать

- Тип уязвимости (например, инъекция, XSS, RCE)
- Полные шаги для воспроизведения
- Воздействие уязвимости
- Любые дополнительные контекстные данные

### Что ожидать

- Ответ в течение 48 часов
- Регулярные обновления о статусе рассмотрения
- Информация о сроке исправления
- Признание вашего вклада (если вы этого хотите)

## Безопасные практики разработки

- Не коммитьте чувствительные данные (ключевые файлы, токены, пароли)
- Проверяйте зависимости на уязвимости
- Следуйте принципу минимальных привилегий

## Обновления безопасности

Обновления безопасности будут в release notes и GitHub security advisories.
