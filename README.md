# Pingcheck с использованием icmplib

## Установка зависимостей

```bash
pip install -r requirements.txt
```

Или просто:

```bash
pip install icmplib
```

## Использование

### Базовый запуск

```bash
python monitor.py --targets ./hosts.txt
```

Если файл не указан, используется `targets.txt` в текущей директории.

### Параметры командной строки

| Параметр | Описание |
|----------|----------|
| `--targets`, `-t` | Путь к файлу со списком целей |
| `--interval`, `-i` | Интервал между пингами (1-1000 секунд, по умолчанию: 1) |
| `--logfile`, `-l` | Путь к файлу лога (например: `logs.txt`) |
| `--only-down`, `-d` | Логировать только события DOWN |

### Управление

Горячие клавиши работают в **английской и русской раскладке**:

| Клавиша | Действие |
|---------|----------|
| `q`, `й`, `Ctrl+C`, `Ctrl+X`, `ф` | Выход из программы |
| `r`, `ц`, `R`, `Ц` | Перечитать файл с целями (без перезапуска) |
| `p`, `к`, `P`, `К` | Пауза/запуск отправки пингов |
| `↑` | Навигация вверх по списку |
| `↓` | Навигация вниз по списку |

### Столбцы таблицы

| Колонка | Описание |
|---------|----------|
| IP Address | IP-адрес или домен |
| Status | UP (доступен) или DOWN (недоступен) |
| RTT avg (ms) | Среднее время отклика за последние 10 попыток |
| RTT last (ms) | Последнее время отклика |
| All-time | Среднее/мин/макс за всё время |
| Loss % | Процент потерянных пакетов (последние 5 попыток) |
| Graph | ASCII-график последних 10 RTT значений |

### ASCII-график RTT

Последний столбец "Graph" отображает историю времени отклика с помощью **Unicode-полублоков**:

| Символ | Значение | Условие |
|--------|----------|---------|
| ` ` (пробел) | Нет данных | 0% (таймаут / ошибка) |
| `▏` | Очень низкий RTT | <10% от максимума |
| `▎` | Низкий RTT | 10–30% |
| `▍` | Средний RTT | 30–50% |
| `▌` | Высокий RTT | 50–70% |
| `▋` | Очень высокий RTT | 70–90% |
| `█` | Критический | >90% |

**Пример:**
```
▏ ▎ ▍ ▌ ▋ █ ▌ ▍ ▎ ▏
```

Такой график мгновенно показывает уровень RTT во времени: пиковые значения и стабильность.

**Примечание:** При пинге 10мс это считается очень низким RTT (показатель `▏`).

### Цветовая схема

| Цвет | Условие |
|------|---------|
| 🟢 Зелёный | RTT avg < 100 мс и loss < 20% |
| 🟡 Жёлтый | RTT avg 100-500 мс или loss 20-50% |
| 🔴 Красный | RTT avg > 500 мс или loss > 50% или недоступен |

### Примеры

```bash
# Запуск с файлом по умолчанию
python monitor.py

# Логирование только событий DOWN
python monitor.py -l logs.txt -d

# Логирование только событий DOWN с интервалом 10 секунд
python monitor.py --interval 10 --logfile logs.txt --only-down

# Запуск с файлом по умолчанию и логированием
python monitor.py -l logs.txt

# Запуск с указанным файлом
python monitor.py --targets ./my-hosts.txt

# Запуск с интервалом 5 секунд и логированием
python monitor.py --interval 5 --logfile ping_log.txt

# Запуск с интервалом 30 секунд
python monitor.py -t ./hosts.txt -i 30

# Запуск с абсолютным путём
python monitor.py -t D:/project/Pingcheck/hosts.txt
```

## Технические детали

- **Язык**: Python 3.8+
- **TUI**: curses (встроенная библиотека)
- **Асинхронность**: asyncio + icmplib
- **Ping**: icmplib (асинхронный ICMP-библиотека)
- **Частота проверки**: 1 раз в секунду для каждой цели (настраивается через `--interval`)
- **Управление**: `p` для паузы/запуска
- **Обновление экрана**: каждые 150мс
- **Ожидание пакета**: 1 секунда
- **Графика**: ASCII-график на основе символов блоков (█) для визуализации RTT истории

## Преимущества icmplib

- ✅ Кроссплатформенная (Windows, Linux, macOS)
- ✅ Не требует `sudo` на Unix-системах
- ✅ Чистый асинхронный API
- ✅ Надёжный разбор результатов
- ✅ Установка через pip

## Требования

- Python 3.8+
- Терминал с поддержкой UTF-8 и **моноширинным шрифтом**
- Библиотека `icmplib` (устанавливается через pip)

**Для корректного отображения ASCII-графика рекомендуется использовать моноширинный шрифт** (например: Consolas, Courier New, Fira Code, JetBrains Mono).

## Примечания

- icmplib использует низкоуровневые ICMP-сокеты
- На некоторых системах может потребоваться запуск от имени администратора для создания ICMP-сокетов
- Программа устойчива к временной недоступности целей
